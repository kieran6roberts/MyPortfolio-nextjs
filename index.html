turncoat brewery

objectives

The aim of this project was to build a full, responsive and fluid site for a ficticious brewing company that included an e-commerce store as the main feature and so I could include the project in my portfolio. It should highlight the products on offer, give the users a contact option and have an easy to use store housing the products.

stack choice

The main feature of the site would be the store which would function with js so that I could practice my skills with this language. Having only just began to learn Node.js I wanted to write a simple server using express and connect it to a database that could store my own custom products without having to reach for a large CMS. I also wanted to practice with another templating language after trying Handlebars so for this project I decided to use EJS.

outcome

Overall this was a successful project as it allowed me to improve my javascript skills by coding the shop and also gain experience in larger applications using tech such as webpack as my build tool.

find a recipe

objectives

Using reactjs, the focus of this project was to build a simple app that would allow users to search for, save and delete recipes. The results should provide enough information for its users to able to go off and cook the meals they find.

stack choice

I decided to use reactjs for this task because I wanted practice using react including concepts such as react-routing, using state and custom hooks. For the styling I went with css modules to keep the css scoped to each component and so I could focus on react fundamentals.

outcome

Although I am happy with the outcome of the project and the things I learned I recognize that I could have implemented an easier solution paticularly with the routing of the app.

search and results pages at mobile device sizes
improvements
  • remove pages such as browse that provide extra clicks for the user to get to the recipe searching
  • add feature for users to save their own recipes
