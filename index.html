turncoat brewery

objectives

The aim of this project was to build a full and responsive site for a ficticious brewing company that included a small e-commerce store as the main feature.

It should highlight the products on offer, give the users a contact option and the store should allow users to add and remove products from a cart while keeping a running total. The items in the cart should also be preserved if the the user navigates to a different page.

stack choice

The main feature of the site is the store which functions using js so that I could practice my skills with this language using many of its core features such as DOM manipulation, iteration and interacting with local storage using the API's provided by the browser.

I wanted to learn how to create my own server and store my products in a database for this project hence I opted for express/mongoose using nodejs to continue my learing of js on both the client-side and server-side. In order to keep my code as dry as possible, I opted to use ejs templating language to reuse partials that were required on several routes.

outcome

Overall this was a successful project as I believe the final product fits the brief of being responsive/modern and allows all of the necessary features requested in the brief to allow the site to function for this ficticious company. Furthermore it was successful in allowing me to gain valuable experience in building a full application and the workflow that comes with it including using build tools, local servers, testing, development and deploying a small full stack application.

home and contact pages at mobile device sizes

I was personally able to learn a considerable amount relating to js best practices such as trying not to repeat myself and code splitting. One thing I will take into future projects is limiting functions to perform one task so that testing and code maintainablity is much easier.

improvements
  • implement checkout process
  • more effective testing (e.g could have used test driven development) leading to simpler refined functions

find a recipe

objectives

Using reactjs, the focus of this project was to build a simple app that would allow users to search for, save and delete recipes. The results should provide enough information using an API, to provide results for its users to able to go off and cook the meals they find. It should provide a clean and simple user-interface using a mobile first approach and ensure it was responsive for all devices

stack choice

I decided to use reactjs for this task because I wanted practice using the library including concepts such as react-routing, using state and custom hooks and I recognized that managing state within the app would be easier by using react. The project is bootstrapped using create-react-app to ensure a fast setup and easy workflow.

For the styling I went with css modules to keep the css scoped to each component and as css was not the focus of this projects I, could focus on react improving my react fundamentals.

outcome

Although I am happy with the outcome of the project and the things I learned I recognize that I could have implemented an easier solution paticularly with the routing of the app.

The app could be built using two routes for search and save and remove unecessary clicks for the user although I wanted to include sub-routing in the app for practice, after all it is a practice project. This would be altered if it was built for a client. using a simple color palette provides the modern and approachable design requested in the brief. The app is limited in the number of search results due to the constraints of the API although it is still able to provide sufficient results to its users.

search and results pages at mobile device sizes
improvements
  • remove pages such as browse that provide extra clicks for the user to get to the recipe searching
  • refactor some larger components (e.g RecipeCard) into collection of smaller reusable components
  • add feature for users to save their own recipes
